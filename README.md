# 定制电商协同 ERP（前端）

## 项目说明
这是一个面向珠宝 / 定制饰品业务的定制电商协同 ERP 前端项目。

当前阶段只优先开发：
- 前端 UI
- 页面结构
- 组件复用
- mock 数据联调

当前阶段**不接真实后端业务流**，先把核心前端主链路跑通。

---

## 当前开发目标

首轮必须完成两大核心操作模块：

1. 商品任务中心
2. 产品管理

并保留两类支撑对象：

3. 交易记录（用于保存一次购买行为的公共信息）
4. 客户（用于后续客户管理与历史查询，首轮先预留数据模型）

首轮必须跑通的核心业务链路：

**产品维护（含规格明细与价格规则）  
→ 商品任务引用产品  
→ 选择规格  
→ 自动带出规格参数  
→ 自动带出基础价格  
→ 叠加固定加价规则  
→ 生成系统参考报价**

---

## 当前业务范围

### 商品任务中心
用于：
- 查看商品任务列表
- 筛选待处理商品任务
- 查看状态、风险、交期
- 在商品任务中引用产品
- 查看来源产品详情
- 选择规格并自动带价
- 推进设计 / 委外 / 生产 / 发货 / 售后等单件商品流程

### 交易记录
用于：
- 保存一次购买行为的公共信息
- 归组同一次购买中的多个商品任务
- 提供整单查询与归档视角

### 产品管理
用于：
- 查看产品列表
- 查看产品详情
- 新建产品
- 编辑产品
- 维护规格明细
- 维护固定加价规则

### 客户（后续接入）
用于：
- 聚合同一客户的多次交易记录
- 查询历史商品任务
- 沉淀客户备注、偏好和售后记录

## 关键业务定义

### 产品（Product）
产品库中的标准模板。

### 客户（Customer）
用于沉淀客户的长期信息、历史交易和历史商品任务。

### 交易记录（TransactionRecord）
一次购买行为的公共信息容器。  
用于保存平台订单号、客户收件信息、支付时间、整单备注等公共信息。

### 商品任务（OrderLine）
系统真正的主操作对象。  
一件商品对应一条独立商品任务，可分别推进规格确认、设计、委外、生产、发货和售后。

### 来源产品（Product Snapshot / Source Product）
商品任务所引用的产品模板快照，用于核对模板值与实际任务值。

### 产品快照
商品任务引用产品时保留的版本化来源信息。

### 系统参考报价
由系统自动计算得到的报价结果。

首轮固定计算公式为：

**系统参考报价 = 规格基础价 + 所有生效固定加价之和**

## 首轮必须支持的业务样例

### 戒指场景
产品中可维护不同圈号的：
- 面宽
- 底宽
- 面厚
- 底厚
- 参考重量
- 基础价格

商品任务引用戒指后：
- 选择 16号
- 自动带出参数
- 自动带出基础价格
- 再叠加材质 / 工艺 / 特殊需求固定加价
- 生成系统参考报价

---

### 吊坠场景
产品中可维护不同档位（小号 / 中号 / 大号）的：
- 长
- 宽
- 厚
- 参考重量
- 基础价格

商品任务引用吊坠后：
- 选择小号
- 自动带出参数
- 自动带出基础价格
- 再叠加工艺 / 特殊需求固定加价
- 生成系统参考报价

- 优先复用组件
- 优先做查看态，再做编辑态

- 不擅自改关键字段语义

## 当前最重要的规则

### 1. 产品与商品任务不是同一个对象
- 产品 = 模板
- 商品任务 = 实例

### 2. 交易记录与商品任务不是同一个对象
- 交易记录 = 一次购买中的公共信息
- 商品任务 = 单件商品的独立执行对象

### 3. 点击商品任务中的商品名称
打开的是：

**来源产品详情**

不是商品任务详情。

### 4. 来源产品详情的作用
用于核对：

- 商品任务参数
- 来源产品原始模板参数

是否一致、已调整或冲突。

### 5. 首轮规格模式
只支持：
- 无规格
- 单轴规格

### 6. 首轮自动报价必须可演示
至少要能演示：
- 戒指自动带价
- 吊坠自动带价

### 7. 物流与售后默认关联商品任务
交易记录页只提供汇总查看与统一入口，不作为物流和售后的唯一归属对象。

## 当前数据结构主线

系统核心对象关系如下：

```text
Customer
  └─ TransactionRecord
       └─ OrderLine
            ├─ ProductSnapshot
            ├─ LogisticsRecord
            ├─ AfterSalesCase
            └─ QuoteResult
```

说明：

- 一个客户可以有多次交易记录
- 一次交易记录可以包含多个商品任务
- 每个商品任务独立推进状态、规格、设计、委外、生产、发货和售后
- 产品始终是模板对象，不直接等于业务执行对象
