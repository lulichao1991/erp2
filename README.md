# 定制电商协同 ERP（前端）

## 项目说明
这是一个面向珠宝 / 定制饰品业务的定制电商协同 ERP 前端项目。

项目当前处于 **Phase 2（业务流转版）**。

当前阶段仍然只优先开发：

- 前端 UI
- 页面结构
- 组件复用
- mock 数据联调
- 页面间业务流转演示

当前阶段**不接真实后端业务流**，先把核心前端主链路跑通，并在 Phase 1 基础上把系统从“前端展示版”升级为“业务流转版”。

---

## 当前开发目标

Phase 2 只优先完成 3 个核心方向：

1. 订单中心升级为业务流转中心
2. 新增任务中心
3. 产品管理升级为产品标准档案

当前必须优先跑通的核心业务链路：

**产品维护（含规格明细、价格规则、业务说明）  
→ 订单引用产品  
→ 选择规格  
→ 自动带出规格参数  
→ 自动带出基础价格  
→ 叠加固定加价规则  
→ 生成系统参考报价  
→ 形成订单商品快照  
→ 发起任务  
→ 推动订单状态流转**

## 当前业务范围

### 产品中心
用于：
- 查看产品列表
- 查看产品详情
- 新建产品
- 编辑产品
- 维护规格明细
- 维护固定加价规则
- 维护业务说明
- 维护图片 / 文件占位信息

### 订单中心
用于：
- 查看订单列表
- 新建订单
- 查看订单详情
- 在订单中引用产品
- 查看来源产品详情
- 选择规格并自动带价
- 展示订单商品快照
- 展示订单状态
- 展示订单时间线
- 从订单发起任务

### 任务中心
用于：
- 查看任务列表
- 查看任务详情
- 跟踪任务状态
- 关联订单
- 必要时关联订单商品
- 支撑订单流转中的执行动作

---

## 关键业务定义

### 产品
产品库中的标准模板。

### 订单商品
订单中的实际商品实例。

### 来源产品
订单商品所引用的产品模板。

### 产品快照
订单引用产品时保留的版本化来源信息。

### 系统参考报价
由系统自动计算得到的报价结果。

当前固定计算公式为：

**系统参考报价 = 规格基础价 + 所有生效固定加价之和**

### 任务
来源于订单流转过程中的业务待办对象。  
当前阶段的任务必须关联订单，必要时关联订单商品。

### 订单时间线
用于记录订单关键动作的追踪信息，例如：
- 创建订单
- 引用产品
- 修改规格
- 重新计算报价
- 状态切换
- 创建任务
- 完成任务
- 更新备注

## 当前订单状态体系

当前订单状态统一为：

- `draft` 草稿
- `pending_confirm` 待确认
- `pending_design` 待设计/建模
- `pending_production_prep` 待生产准备
- `pending_shipping` 待发货
- `after_sales` 售后中
- `completed` 已完成
- `cancelled` 已取消

当前阶段只做轻量流转逻辑，不做复杂审批流和完整工厂协同。

---

当前阶段任务类型统一为：

- `design_modeling`
- `production_prep`

当前任务主要服务于订单流转，不扩展为通用协作平台。

## 当前必须支持的业务样例

### 戒指场景
产品中可维护不同圈号的：
- 面宽
- 底宽
- 面厚
- 底厚
- 参考重量
- 基础价格

订单引用戒指后：
- 选择 16 号
- 自动带出参数
- 自动带出基础价格
- 再叠加材质 / 工艺 / 特殊需求固定加价
- 生成系统参考报价
- 形成订单商品快照
- 可继续发起任务

---

### 吊坠场景
产品中可维护不同档位（小号 / 中号 / 大号）的：
- 长
- 宽
- 厚
- 参考重量
- 基础价格

订单引用吊坠后：
- 选择小号
- 自动带出参数
- 自动带出基础价格
- 再叠加工艺 / 特殊需求固定加价
- 生成系统参考报价
- 形成订单商品快照
- 可继续发起任务

## 当前阶段不做

Phase 2 暂不处理以下能力：

- 真实后端接口联调
- 多轴规格
- 复杂报价公式
- 报价审批流
- 多版本报价历史
- 财务中心
- 工厂协同中心
- 复杂物流系统
- 完整售后流程系统
- 完整权限系统
- 文件上传高级管理
- BI 报表
- 小程序端

---

## 当前开发原则

- 前端优先
- 主链路优先
- 小步迭代
- 优先复用组件
- 优先增量修改
- 优先做查看态，再做编辑态
- 优先做骨架，再做复杂交互
- 不擅自改业务定义
- 不擅自改关键字段语义
- 不提前扩展到与本期无关的中心模块

---

## 当前最重要的规则

### 1. 产品与订单商品不是同一个对象
- 产品 = 模板
- 订单商品 = 实例

### 2. 点击订单中的商品名称
打开的是：

**来源产品详情**

不是订单商品详情。

### 3. 来源产品详情的作用
用于核对：

- 订单商品参数
- 来源产品原始模板参数

是否一致、已调整或冲突。

### 4. 当前规格模式
只支持：
- 无规格
- 单轴规格

### 5. 自动报价必须持续可演示
至少要能演示：
- 戒指自动带价
- 吊坠自动带价

### 6. 任务必须有业务来源
当前阶段任务必须来源于订单，必要时来源于订单商品。

### 7. 订单时间线必须可追溯
订单中的关键动作必须能在时间线中体现。
