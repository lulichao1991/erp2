# 定制电商协同 ERP（前端）

## 项目说明
这是一个面向珠宝 / 定制饰品业务的定制电商协同 ERP 前端项目。

当前阶段只优先开发：
- 前端 UI
- 页面结构
- 组件复用
- mock 数据联调

当前阶段**不接真实后端业务流**，先把核心前端主链路跑通。

---

## 当前开发目标

首轮必须完成两大核心模块：

1. 订单中心
2. 产品管理

首轮必须跑通的核心业务链路：

**产品维护（含规格明细与价格规则）  
→ 订单引用产品  
→ 选择规格  
→ 自动带出规格参数  
→ 自动带出基础价格  
→ 叠加固定加价规则  
→ 生成系统参考报价**

---

## 当前业务范围

### 订单中心
用于：
- 查看订单列表
- 新建订单
- 查看订单详情
- 在订单中引用产品
- 查看来源产品详情
- 选择规格并自动带价

### 产品管理
用于：
- 查看产品列表
- 查看产品详情
- 新建产品
- 编辑产品
- 维护规格明细
- 维护固定加价规则

---

## 关键业务定义

### 产品
产品库中的标准模板。

### 订单商品
订单中的实际商品实例。

### 来源产品
订单商品所引用的产品模板。

### 产品快照
订单引用产品时保留的版本化来源信息。

### 系统参考报价
由系统自动计算得到的报价结果。

首轮固定计算公式为：

**系统参考报价 = 规格基础价 + 所有生效固定加价之和**

## 首轮必须支持的业务样例

### 戒指场景
产品中可维护不同圈号的：
- 面宽
- 底宽
- 面厚
- 底厚
- 参考重量
- 基础价格

订单引用戒指后：
- 选择 16号
- 自动带出参数
- 自动带出基础价格
- 再叠加材质 / 工艺 / 特殊需求固定加价
- 生成系统参考报价

---

### 吊坠场景
产品中可维护不同档位（小号 / 中号 / 大号）的：
- 长
- 宽
- 厚
- 参考重量
- 基础价格

订单引用吊坠后：
- 选择小号
- 自动带出参数
- 自动带出基础价格
- 再叠加工艺 / 特殊需求固定加价
- 生成系统参考报价

---

## 当前阶段不做

首轮暂不处理以下能力：

- 真实后端接口联调
- 多轴规格
- 复杂报价公式
- 人工最终报价覆盖系统价
- 报价审批流
- 多版本报价历史
- 复杂物流系统
- 售后复杂流转
- 文件上传高级管理
- BI 报表
- 小程序端

## 当前开发原则

- 前端优先
- 主链路优先
- 小步迭代
- 优先复用组件
- 优先做查看态，再做编辑态
- 优先做骨架，再做复杂交互
- 不擅自改业务定义
- 不擅自改关键字段语义

---

## 当前最重要的规则

### 1. 产品与订单商品不是同一个对象
- 产品 = 模板
- 订单商品 = 实例

### 2. 点击订单中的商品名称
打开的是：

**来源产品详情**

不是订单商品详情。

### 3. 来源产品详情的作用
用于核对：

- 订单商品参数
- 来源产品原始模板参数

是否一致、已调整或冲突。

### 4. 首轮规格模式
只支持：
- 无规格
- 单轴规格

### 5. 首轮自动报价必须可演示
至少要能演示：
- 戒指自动带价
- 吊坠自动带价
