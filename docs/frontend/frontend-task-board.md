# 前端任务板（首轮）

## 1. 文档目标
本文件用于把首轮前端开发任务按阶段拆开，方便：

- 你自己排开发顺序
- AI / Codex 按阶段执行
- 每轮验收时知道“做到哪一步才算完成”

首轮任务只围绕两大模块展开：

1. 商品任务中心
2. 产品管理

并且必须跑通这条主链路：

**产品维护（含规格明细与固定加价规则）  
→ 商品任务引用产品  
→ 选择规格  
→ 自动带出规格参数  
→ 自动带出基础价格  
→ 叠加固定加价规则  
→ 生成系统参考报价**

---

## 2. 首轮总原则

### 2.1 先主骨架，再关键交互
优先顺序固定为：

1. 全局框架
2. 产品查看页
3. 产品编辑最小版
4. 交易记录骨架
5. 产品引用
6. 自动带价

---

### 2.2 先查看态，再编辑态
首轮不是把所有编辑能力都做满，而是优先保证：

- 页面结构清楚
- 核心交互能演示
- 业务主链路能走通

---

### 2.3 自动带价必须是首轮必做
自动带价不是增强项，不能后置。

---

## 13. 2026-04-23 客服角色字段补充待办

当进入“按角色补订单详情”阶段时，客服角色优先补以下字段，不要只按现有 `基础信息区 / 客户信息区 / 时间信息区` 的卡片名称判断完成度。

### 13.1 订单级字段
- `接待客服`
  - 复用当前 `客服负责人` 语义
  - 优先调整展示文案
- `订单登记时间`
  - 新建订单时自动生成
  - 订单详情中可查看
- `平台ID`
  - 与客户姓名并列展示
- `订单类型`
  - 统一区分：
    - 销售订单
    - 内部订单
- `来源渠道`
  - 做成可维护字典
- `平台订单编号`
  - 保持现有命名
  - 与 `订单类型 / 来源渠道` 配合展示

### 13.2 订单类型对应权限规则
- `销售订单`
  - 可由客服、管理层、运营等角色填写
- `内部订单`
  - 仅允许设计、管理员填写
- `销售` 不再单独建角色，直接并入 `客服`
- 后续前端权限要按 `角色 + 订单类型` 联合判断，而不只按角色判断

### 13.3 时间字段
- `客户期望时间`
- `计划交期`
- `承诺交期`

要求：
- 计划交期支持根据工期逻辑自动推导
- 需要校验三者是否冲突

### 13.4 财务信息
- `系统参考报价` 只保留为系统自动计算出的参考价
- 不等同于客服实际填写价格
- 财务区不能只靠 3 个静态字段，需要同时做：
  - 财务摘要字段
  - 财务流水记录

#### 财务摘要字段
- `系统参考价`
- `实际成交价`
- `定金`
- `尾款`
- `累计收款`
- `累计退款`
- `累计净收款`
- `是否开过发票`
- `财务备注`

#### 财务流水类型
- `定金收款`
- `尾款收款`
- `平台退款`
- `线下退款 / 非平台退款`
- `售后补款`
- `售后退款`

#### 说明
- 用户给出的案例已经说明，一个订单可能经历多次收款与多次退款
- 所以后续订单详情里的财务区应优先做“摘要 + 流水”双层结构
- 不建议只停留在 `定金 / 尾款 / 退差补偿` 三个输入框
- 页面文案优先采用：
  - `实际成交价`
  - `累计净收款`

### 13.5 推荐执行顺序
1. 先补订单级字段：`平台ID / 来源渠道 / 订单登记时间 / 计划交期`
2. 再把 `订单类型` 收口为 `销售订单 / 内部订单`
3. 再补 `平台订单编号`
4. 再补财务摘要字段与财务流水
5. 最后扩展商品级客服采集字段与售后信息
