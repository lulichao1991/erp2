# 页面树与路由结构（Purchase + OrderLine）

## 1. 文档目标
本文件用于明确当前前端页面树、主路由、兼容路由边界，以及页面之间的跳转关系。

当前项目主线对象是：

- `Customer`：客户
- `Purchase`：购买记录 / 一次购买公共信息
- `OrderLine`：商品行 / 一件商品的执行对象
- `Product`：产品模板
- `ProductSnapshot`：来源产品快照

当前系统主操作对象是 **OrderLine（商品行）**。

---

## 2. 路由设计原则

### 2.1 当前主入口以商品行为主
商品行中心是当前主工作台，一行代表一件商品。

购买记录只是归组对象，用于查看一次购买中的公共信息和本次购买下的所有商品行。

### 2.2 产品模板和商品行必须分开
产品页展示 `Product` 模板。

商品行引用产品时保留 `ProductSnapshot`，用于核对来源模板与本次商品行实际选择。

### 2.3 高频动作优先用弹窗 / 抽屉承接
以下能力优先挂在上下文里，不急着拆成一级页面：

- 来源产品详情
- 物流记录
- 售后记录
- 商品行编辑

### 2.4 legacy `/orders` 已删除
旧 `/orders` 不再是当前主入口，也不再作为可访问兼容路由保留。

当前新主流程入口是 `/order-lines` 与 `/purchases/new`：
- 商品行执行、筛选、物流、售后和生产推进走 `/order-lines`
- 新建一次购买和多条商品行草稿走 `/purchases/new`
- 查看一次购买的归组信息走 `/purchases/:purchaseId`

删除准备记录见：`docs/frontend/legacy-orders-removal-plan.md`。

## 3. 一级导航结构

当前一级导航展示：

- 工作台
- 商品行中心
- 客户中心
- 任务中心
- 产品管理
- 生产计划（仅工厂角色可见）

说明：
- 导航中只展示“商品行中心”
- 不再同时展示“商品任务中心”或“订单中心”
- `/orders` 旧模块已删除，不作为侧边栏入口展示
- `/production-plan` 仍可作为工厂角色入口展示，但其视图数据只来自当前 `tasks + purchases + orderLines + products`

### `/customers`
客户中心。

用途：
- 查看客户基础信息
- 查看当前系统内购买记录、商品行和售后聚合
- 进入客户详情

说明：
- 第一版为只读轻量聚合
- 不做复杂 CRM、营销自动化或客户画像

### `/customers/:customerId`
客户详情。

用途：
- 查看客户基础信息
- 查看历史购买记录、历史商品行和售后摘要
- 跳转到购买记录详情或商品行中心

说明：
- 当前聚合统计与外部历史 totals 分开展示
- 不把客户中心扩展成执行工作台

## 4. 当前主页面树

### `/order-lines`
商品行中心。

用途：
- 搜索商品行
- 筛选商品行
- 查看状态、风险、交期
- 查看当前负责人
- 查看商品行编号、生产任务编号、款式、版本、货号 / SKU
- 独立推进设计、生产、发货与售后
- 打开商品行详情抽屉
- 进入所属购买记录

说明：
- 本页一行代表一件商品
- 不是一行代表一笔购买记录
- 状态筛选和待办判断优先基于 `OrderLine.lineStatus`
- 商品行详情抽屉展示客服资料完整度，并支持客服确认分流
- `OrderLine.status` 短期保留为兼容展示字段，新任务分组和后续角色视图不要继续扩大它的主流程用途
- 当前商品行字段用于支撑后续跟单、设计 / 建模、工厂协同和财务核算视图

---

### `/purchases/new`
新建购买记录。

用途：
- 创建 `Purchase` 草稿
- 录入一次购买的公共信息
- 新增多条商品行草稿
- 每条商品行草稿可独立引用产品模板
- 选择规格并自动生成系统参考报价
- 录入商品行编号、生产任务编号、货号 / SKU、款式、版本、材质、尺寸、工艺、印记和特殊需求
- 标记是否加急、是否需要设计、是否需要建模、是否需要出蜡
- 展示资料完整度和缺失项
- 客服确认完成后，按设计 / 建模需求分流到对应 `lineStatus`

说明：
- 本页创建的是购买记录
- 页面中的每张商品行草稿卡代表一条未来的 `OrderLine`
- 不把单件商品执行字段塞进 `Purchase`

---

### `/purchases/:purchaseId`
购买记录详情。

用途：
- 查看购买记录公共信息
- 查看客户与收件信息
- 查看付款汇总
- 查看本次购买下的所有商品行
- 从商品行打开详情抽屉
- 查看购买记录时间线

说明：
- 本页是购买归组页
- 商品行才是具体执行对象
- 物流和售后记录默认归属 `orderLineId`

---

### `/products`
产品列表页。

用途：
- 搜索产品模板
- 筛选产品模板
- 查看是否可引用
- 查看价格摘要
- 进入产品详情页
- 进入产品编辑页

---

### `/products/new`
产品新建页。

用途：
- 创建新产品模板
- 录入基础信息
- 配置规格明细
- 配置固定加价规则
- 维护生产参考与文件资料

---

### `/products/:productId`
产品详情页。

用途：
- 查看产品模板原始资料
- 查看参数配置
- 查看价格规则
- 查看定制规则
- 查看生产参考
- 查看图片与文件

说明：
- 这里展示的是产品模板，不是商品行实例

---

### `/products/:productId/edit`
产品编辑页。

用途：
- 修改产品模板
- 修改规格明细
- 修改加价规则
- 保存产品信息

---

## 5. 上下文弹窗与抽屉

### 商品行详情抽屉
建议挂载在 `/order-lines` 或 `/purchases/:purchaseId` 上下文中。

用途：
- 查看商品行详情
- 编辑商品行基础信息 / 实际需求
- 更新设计、委外、生产信息
- 新增或编辑物流记录
- 新增或编辑售后记录

---

### 来源产品详情抽屉
建议挂载在商品行上下文中。

用途：
- 查看来源产品原始资料
- 核对商品行参数与来源产品参数差异
- 明确 `Product` 是模板，`OrderLine` 是实例

---

### 物流记录
说明：
- 物流记录默认关联 `orderLineId`
- 购买记录页只提供汇总查看与统一入口

---

### 售后记录
说明：
- 售后记录默认关联 `orderLineId`
- 同一次购买中的单件商品可以独立进入售后

---

## 6. 页面关系图

```text
商品行中心 /order-lines
  ├─ 商品行详情抽屉
  ├─ 来源产品详情抽屉
  └─ 所属购买记录 /purchases/:purchaseId

购买记录 /purchases
  ├─ 购买记录入口 /purchases → /purchases/new
  ├─ 新建购买记录 /purchases/new
  └─ 购买记录详情 /purchases/:purchaseId
       ├─ 商品行列表
       ├─ 商品行详情抽屉
       ├─ 物流记录入口
       └─ 售后记录入口

产品管理 /products
  ├─ 新建产品 /products/new
  ├─ 产品详情 /products/:productId
  └─ 产品编辑 /products/:productId/edit

legacy /orders
  └─ 已删除；旧实现仅可通过 git 历史回看
```

## 8. 路由最小集

当前主线必须保持可访问：

```text
/order-lines
/purchases
/purchases/new
/purchases/:purchaseId
/products
/products/new
/products/:productId
/products/:productId/edit
```

legacy `/orders` 路由已删除；不再列入可访问路由最小集。

---

## 9. 暂不复杂化的内容

当前清理收口阶段暂不新增：

- 物流独立列表页
- 售后独立列表页
- 财务中心
- 完整工厂协同系统
- 产品引用选择器独立页面
- 来源产品详情独立页面
- 复杂权限系统

---

## 10. 验收标准

- 侧边栏只展示“商品行中心”，不同时展示“商品任务中心”或“订单中心”
- `/order-lines` 可作为当前商品行主入口访问
- `/purchases/new` 可作为新建购买记录入口访问
- `/purchases/:purchaseId` 可作为购买记录详情入口访问
- `/products` 可作为产品管理入口访问
- `/production-plan` 可作为 current-only 生产计划入口访问
- `/orders` 不再可访问，当前主流程不依赖旧订单模块
- 文档中不再把 `TransactionRecord` 作为当前主模型
- 文档中不再把 `/orders` 写成当前主模块
