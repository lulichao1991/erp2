# 页面树与路由结构草案（商品任务中心 + 产品管理）

## 1. 文档目标
本文件用于明确前端首轮页面树、主路由、上下文弹窗 / 抽屉挂载方式，以及页面之间的跳转关系。

当前首轮聚焦三个对象：

- 商品任务
- 交易记录
- 产品模板

并围绕这条主链路组织页面关系：

**产品维护  
→ 商品任务引用产品  
→ 查看来源产品  
→ 选择规格  
→ 自动带价**

---

## 2. 路由设计原则

### 2.1 主页面尽量少
首轮只保留真正的主工作台页面，不要把所有能力都拆成独立页面。

### 2.2 高频动作优先用弹窗 / 抽屉承接
例如：

- 产品引用选择器
- 来源产品详情
- 物流记录
- 售后记录
- 商品任务编辑

这些优先挂在上下文里，不急着拆成一级页面。

### 2.3 商品任务与产品的主页面要能互相跳转
- 商品任务中可以查看来源产品
- 产品中后续可查看引用记录（首轮可先预留）

### 2.4 路由先保留现有路径
首轮暂沿用现有 `orders` 路由，避免页面和代码一次性大改。

也就是：

- `/orders` 实际语义为商品任务中心
- `/orders/new` 实际语义为新建交易记录页
- `/orders/:orderId` 实际语义为交易记录详情页

### 2.5 路由要保留上下文
商品任务页或交易记录页打开弹窗 / 抽屉时，建议通过 query 表达状态，避免页面刷新后上下文丢失。

## 4. 商品任务模块页面树

## 4.1 主页面

### `/orders`
商品任务中心

用途：
- 搜索商品任务
- 筛选商品任务
- 查看状态、风险、交期
- 查看当前负责人
- 进入交易记录详情页
- 后续支持直接打开商品任务详情抽屉

说明：
- 本页一行代表一件商品任务
- 不是一行代表一笔交易记录

---

### `/orders/new`
新建交易记录页

用途：
- 创建交易记录
- 录入公共信息
- 新增多个商品任务
- 在商品任务卡中引用产品
- 保存草稿

说明：
- 本页创建的是交易记录
- 页面中的商品卡代表多个独立商品任务

---

### `/orders/:orderId`
交易记录详情页

用途：
- 查看交易记录概览
- 查看交易级公共信息
- 查看本次交易下的商品任务卡组
- 从商品任务中查看来源产品
- 进行规格选择与自动报价
- 查看物流 / 售后 / 附件 / 日志摘要

说明：
- 本页是“交易归组页”
- 商品任务才是具体执行对象

---

## 4.2 商品任务 / 交易上下文中的弹窗与抽屉

这些能力建议不单独建一级页面，而挂载在商品任务中心或交易记录详情页上下文里。

### 产品引用选择器
建议挂载为：

- `/orders/new?modal=product-picker&itemKey=temp-1`
- `/orders/:orderId?modal=product-picker&itemId=xxx`

用途：
- 在商品任务卡中引用产品模板

---

### 来源产品详情抽屉
建议挂载为：

- `/orders/:orderId?drawer=source-product&itemId=xxx`

用途：
- 查看来源产品原始资料
- 核对商品任务参数与来源产品参数差异

---

### 物流记录弹窗（后补）
建议挂载为：

- `/orders/:orderId?modal=add-logistics&itemId=xxx`
- `/orders/:orderId?modal=edit-logistics&itemId=xxx&logisticsId=xxx`

说明：
- 物流记录默认关联商品任务
- 交易记录页仅提供统一上下文入口

---

### 售后记录弹窗（后补）
建议挂载为：

- `/orders/:orderId?modal=add-aftersales&itemId=xxx`
- `/orders/:orderId?modal=edit-aftersales&itemId=xxx&aftersalesId=xxx`

说明：
- 售后记录默认关联商品任务
- 支持同一交易中的单件商品单独进入售后

---

### 商品任务编辑抽屉（后补）
后续可预留：

- `/orders/:orderId?drawer=edit-item-spec&itemId=xxx`
- `/orders/:orderId?drawer=edit-design-info&itemId=xxx`
- `/orders/:orderId?drawer=edit-outsource-info&itemId=xxx`
- `/orders/:orderId?drawer=edit-production-info&itemId=xxx`

首轮可先不实现，但结构上要留空间。

建议挂载为：

## 7. 商品任务与产品之间的桥接关系

### 7.1 从商品任务进入产品

#### 入口 1

商品任务卡点击：

- 商品名称

动作：

- 打开来源产品详情抽屉

#### 入口 2

来源产品详情抽屉中点击：

- 在产品模块中查看

动作：

- 跳转 `/products/:productId`

------

### 7.2 从产品进入商品任务 / 交易记录

首轮可以先不做强跳转，但后续建议支持：

- 在产品引用记录中点击交易编号
- 跳转 `/orders/:orderId`

后续若商品任务有独立详情抽屉，也可支持：

- 从产品引用记录中打开指定 `itemId` 的商品任务上下文

## 8. 页面入口说明

## 8.1 商品任务中心入口

- 侧边栏：商品任务中心
- 其他页面返回入口

## 8.2 新建交易记录页入口

- 商品任务中心右上角：新建交易记录

## 8.3 交易记录详情页入口

- 从商品任务中心点击某一行对应交易记录
- 后续可从工作台卡片进入

## 8.4 产品列表页入口

- 侧边栏：产品管理
- 商品任务页引用产品时不会直接跳转这里，而是优先用产品引用选择器

## 8.5 产品详情页入口

- 产品列表点击产品
- 来源产品详情抽屉中点击“在产品模块中查看”

## 8.6 产品编辑页入口

- 产品列表操作列点击编辑
- 产品详情页右上角点击编辑产品
- 产品列表右上角点击新建产品进入新建页

## 11. 首轮暂不复杂化的内容

为了避免首轮路由过度复杂，下面这些暂不做成单独页面：

- 物流单独列表页
- 售后单独列表页
- 客户中心独立页面
- 交易记录独立列表页
- 产品引用选择器独立页面
- 来源产品详情独立页面
- 交易记录详情页内部 tab 子路由
- 产品详情页内部 tab 子路由

## 12. 前端目录建议（页面层）

如果使用 React Router 或类似结构，建议页面目录先按下面组织：

```text
pages/
  orders/
    index
    new
    [orderId]/
      index
  products/
    index
    new
    [productId]/
      index
      edit
```

说明：

- 首轮先保留 `orders` 目录名，减少改动范围
- 但页面语义统一按“商品任务中心 / 交易记录详情”理解

弹窗 / 抽屉建议放在对应 feature 目录，而不是作为独立页面文件强行挂出去。

## 13. 首轮验收标准

### 13.1 路由清晰

用户能从侧边栏进入商品任务中心和产品管理，并按预期跳转。

### 13.2 页面关系稳定

- 商品任务 → 产品引用选择器
- 商品任务 → 来源产品详情抽屉
- 来源产品详情抽屉 → 产品详情页

这条链路必须明确。

### 13.3 不过度拆页面

首轮页面数量控制合理，主要依赖主页面 + 上下文弹窗 / 抽屉完成操作。

### 13.4 后续可扩展

物流、售后、客户、交易记录、文件、版本等后续能力能自然接到现有页面树上，不需要推翻重建。

### 13.5 商品行为主的口径稳定

文档中不得再出现“整单是唯一主操作对象”的表述。
商品任务是系统真正的主操作对象，交易记录是归组与查询对象。

## 这版重点修正了什么

最重要的改动有 5 个：

1. 把一级导航的语义从“订单中心”改成“商品任务中心”
2. 保留 `/orders` 路由，但把它正式定义为“商品任务中心”
3. 把 `/orders/new` 改成“新建交易记录页”
4. 把 `/orders/:orderId` 改成“交易记录详情页”
5. 把物流和售后的挂载方式改成 **默认关联 `itemId`**

这几处正好对应你当前旧文档里最容易把系统带回“整单为主”的地方。你原来的路由文档里，物流和售后弹窗还是整单语义，订单详情页也是主协同页；但 UI 文档已经写了每个商品都是独立商品卡，所以现在这样改是顺着现有胚子继续推。:contentReference[oaicite:2]{index=2} :contentReference[oaicite:3]{index=3}
