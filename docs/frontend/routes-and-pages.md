# 页面树与路由结构（Purchase + OrderLine）

## 1. 文档目标
本文件用于明确当前前端页面树、主路由、兼容路由边界，以及页面之间的跳转关系。

当前项目主线对象是：

- `Customer`：客户
- `Purchase`：购买记录 / 一次购买公共信息
- `OrderLine`：商品行 / 一件商品的执行对象
- `Product`：产品模板
- `ProductSnapshot`：来源产品快照

当前系统主操作对象是 **OrderLine（商品行）**。

---

## 2. 路由设计原则

### 2.1 当前主入口以商品行为主
商品行中心是当前主工作台，一行代表一件商品。

购买记录只是归组对象，用于查看一次购买中的公共信息和本次购买下的所有商品行。

### 2.2 产品模板和商品行必须分开
产品页展示 `Product` 模板。

商品行引用产品时保留 `ProductSnapshot`，用于核对来源模板与本次商品行实际选择。

### 2.3 高频动作优先用弹窗 / 抽屉承接
以下能力优先挂在上下文里，不急着拆成一级页面：

- 来源产品详情
- 物流记录
- 售后记录
- 商品行编辑

### 2.4 legacy `/orders` 已删除
旧 `/orders` 不再是当前主入口，也不再作为可访问兼容路由保留。

当前新主流程入口是 `/order-lines` 与 `/purchases/new`：
- 商品行执行、筛选、物流、售后和生产推进走 `/order-lines`
- 新建一次购买和多条商品行草稿走 `/purchases/new`
- 查看一次购买的归组信息走 `/purchases/:purchaseId`

删除准备记录见：`docs/frontend/legacy-orders-removal-plan.md`。

## 3. 一级导航结构

当前一级导航展示：

- 工作台
- 商品行中心
- 客户中心
- 任务中心
- 生产跟进
- 设计建模
- 工厂协同（仅工厂角色可见）
- 财务中心
- 仓库商品
- 管理看板
- 产品管理
- 生产计划（仅工厂角色可见）

说明：
- 导航中只展示“商品行中心”
- 不再同时展示“商品任务中心”或“订单中心”
- `/orders` 旧模块已删除，不作为侧边栏入口展示
- 侧边栏会根据当前前端 mock role capability 显示可见入口；这不是后端权限系统
- `/production-plan` 仍可作为工厂角色入口展示，但其视图数据只来自当前 `tasks + purchases + orderLines + products`
- `/production-follow-up` 是跟单生产推进入口，视图数据只来自当前 `OrderLine` 和所属 `Purchase`
- `/design-modeling` 是设计 / 建模工作台，视图数据只来自当前 `OrderLine`
- `/factory` 是工厂协同中心，只展示分配给当前 mock 工厂的 `OrderLine` 生产信息
- `/finance` 是财务中心，围绕 `Purchase.finance` 和 `OrderLine` 成本 / 工厂回传数据做确认
- `/inventory` 是仓库商品管理，围绕 `InventoryItem` 记录设计留样、客户退货、常备采购和其他库存
- `/management` 是管理看板，围绕 `Purchase + OrderLine` 汇总业务、生产风险、财务、角色负载和工厂表现

角色入口口径：
- 客服可见客户、商品行、任务和产品相关入口
- 跟单可见生产跟进、生产计划、商品行和任务入口
- 设计 / 建模可见设计建模、商品行、任务和产品入口
- 工厂可见工厂协同和生产计划入口
- 库管可见仓库商品、商品行和产品入口
- 财务可见财务中心和商品行入口
- 管理可见管理看板和各业务观察入口
- 管理员可见全部前端入口

说明：
- 当前聚合统计与外部历史 totals 分开展示
- 不把客户中心扩展成执行工作台

## 4. 当前主页面树

### `/order-lines`
商品行中心。

用途：
- 搜索商品行
- 筛选商品行
- 查看状态、风险、交期
- 查看当前负责人
- 查看商品行编号、生产任务编号、款式、版本、货号 / SKU
- 独立推进设计、生产、发货与售后
- 打开商品行详情抽屉
- 进入所属购买记录

说明：
- 本页一行代表一件商品
- 不是一行代表一笔购买记录
- 状态筛选和待办判断优先基于 `OrderLine.lineStatus`
- 商品行详情抽屉展示客服资料完整度，并支持客服确认分流
- `OrderLine.status` 短期保留为兼容展示字段，新任务分组和后续角色视图不要继续扩大它的主流程用途
- 当前商品行字段用于支撑后续跟单、设计 / 建模、工厂协同和财务核算视图

---

### `/purchases/new`
新建购买记录。

用途：
- 创建 `Purchase` 草稿
- 录入一次购买的公共信息
- 新增多条商品行草稿
- 每条商品行草稿可独立引用产品模板
- 选择规格并自动生成系统参考报价
- 录入商品行编号、生产任务编号、货号 / SKU、款式、版本、材质、尺寸、工艺、印记和特殊需求
- 标记是否加急、是否需要设计、是否需要建模、是否需要出蜡
- 展示资料完整度和缺失项
- 客服确认完成后，按设计 / 建模需求分流到对应 `lineStatus`

说明：
- 本页创建的是购买记录
- 页面中的每张商品行草稿卡代表一条未来的 `OrderLine`
- 不把单件商品执行字段塞进 `Purchase`

---

### `/purchases/:purchaseId`
购买记录详情。

用途：
- 查看购买记录公共信息
- 查看客户与收件信息
- 查看付款汇总
- 查看本次购买下的所有商品行
- 从商品行打开详情抽屉
- 查看购买记录时间线

说明：
- 本页是购买归组页
- 商品行才是具体执行对象
- 物流和售后记录默认归属 `orderLineId`

---

### `/production-follow-up`
生产跟进 / 跟单视图。

用途：
- 按商品行查看待跟单审核、待下发生产、生产中、待工厂回传、异常 / 逾期
- 分配工厂、设置工厂计划交期
- 标记资料已齐、下发生产、标记生产中、标记阻塞
- 退回客服补资料或退回设计 / 建模修改

说明：
- 本页只基于 `OrderLine` 的 `lineStatus / productionStatus / factoryStatus` 做生产推进
- 购买记录只用于展示 `purchaseNo` 和跳转归组页
- 不恢复旧 `/orders` 或旧订单单件逻辑

---

### `/design-modeling`
设计 / 建模工作台。

用途：
- 查看待设计、设计中、待建模、建模中、待修改和已完成商品行
- 领取设计 / 建模任务
- 标记设计中、设计完成、建模中、建模完成
- 记录出蜡文件和发送出蜡厂时间
- 记录设计 / 建模备注和修改原因

说明：
- 本页只基于 `OrderLine` 的设计、建模和出蜡字段
- 不展示客户联系方式、销售金额、定金、尾款或利润
- 不接真实文件上传，第一版使用 mock file URL / file name 记录

---

### `/factory`
工厂协同中心。

用途：
- 查看分配给当前 mock 工厂的商品行生产任务
- 接收任务、标记开始生产、标记生产完成
- 填写重量、材质、石料、工费、成品图和结算单等生产回传数据
- 提交给跟单 / 财务确认
- 标记工厂异常并填写工厂备注

说明：
- 本页只基于 `OrderLine.factoryId / productionStatus / factoryStatus / productionData`
- 不展示客户姓名、联系方式、收货地址、销售价格、定金、尾款、利润或财务备注
- 当前为前端 mock 角色与 mock 数据，不接真实后端鉴权或真实上传

---

### `/finance`
财务中心。

用途：
- 查看购买记录收款汇总、定金和尾款状态
- 查看工厂回传的重量、材质、工费和结算资料
- 确认尾款、确认工厂结算金额
- 标记财务异常、填写财务备注、锁定财务数据

说明：
- 本页围绕 `Purchase.finance` 与 `OrderLine.financeStatus / productionData`
- 财务可以查看金额和成本，但不负责推进设计、建模或生产状态
- 当前不做真实导出、后端收款流水或复杂财务审批

---

### `/inventory`
仓库商品管理。

用途：
- 查看设计留样、客户退货、常备采购、寄售和其他库存
- 通过全部库存、设计留样、客户退货、待检 / 瑕疵、已占用、低库存、不可用快捷视图定位重点库存
- 按来源、库存状态、成色、库位和关键词筛选库存商品
- 查看库存与 Product / Purchase / OrderLine / Customer 的来源关联
- 查看单条库存详情和该库存自己的流转记录
- 查看关联商品行的库存占用、释放、出库追溯摘要
- 查看总数、可用数量、已占用数量、库位、入库时间和库管信息
- 对客户退货、待检修和瑕疵库存做质检处置，更新成色、状态、可用数量和库位
- 做前端 mock 库存盘点，按实盘总数和实盘可用数调整库存台账，并生成调整流水
- 登记前端 mock 入库、占用、释放、出库、报废和库位调整，并可把流转记录关联到具体商品行
- 按流转类型、关联商品行和关键词筛选库存流转记录

说明：
- 本页基于 `InventoryItem` 做库存资产台账
- 库存记录可以关联 `Product / Purchase / OrderLine / Customer`，但不替代产品模板或商品行执行流
- 质检处置只更新库存资产和库存流转记录，不推进商品行生产、财务或售后状态
- 库存盘点只更新库存数量、可用数量、库位和库存流转记录，不推进商品行生产、财务或售后状态
- 关联商品行的库存占用 / 出库只作为库存追溯，不改写 `OrderLine.lineStatus`
- 当前为前端 mock 版本，不做真实审批、盘点、条码系统或后端库存锁定

- 查看 Purchase 和 OrderLine 业务总览

- 管理看板不承接客服、跟单、设计、工厂或财务录入动作
- 统计逻辑集中在 management selector 中，不堆在 JSX
- 生产风险、资料完整性、财务风险等规则优先复用 `orderLineRiskSelectors`

## 10. 验收标准

- 侧边栏只展示“商品行中心”，不同时展示“商品任务中心”或“订单中心”
- `/order-lines` 可作为当前商品行主入口访问
- `/purchases/new` 可作为新建购买记录入口访问
- `/purchases/:purchaseId` 可作为购买记录详情入口访问
- `/products` 可作为产品管理入口访问
- `/production-plan` 可作为 current-only 生产计划入口访问
- `/orders` 不再可访问，当前主流程不依赖旧订单模块
- 文档中不再把 `TransactionRecord` 作为当前主模型
- 文档中不再把 `/orders` 写成当前主模块
