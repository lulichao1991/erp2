# 页面结构总览（UI Structure）

## 1. 文档目标
本文件用于统一首轮前端页面结构，明确：

- 每个主页面从上到下有哪些区块
- 区块之间的层级关系
- 哪些区块是首轮必做
- 哪些区块首轮先做占位

当前只覆盖两个模块：

- 订单中心
- 产品管理

并重点服务于这条首轮业务链：

**产品维护  
→ 订单引用产品  
→ 选择规格  
→ 自动带出参数  
→ 自动带出价格  
→ 查看来源产品详情**

---

## 2. 总体 UI 原则

### 2.1 页面不是大表单
订单详情页和产品详情页都不应做成传统的超长表单。

### 2.2 先看全局，再看分区
页面结构优先遵循：

- 顶部摘要
- 中部核心信息区
- 下部辅助记录区

### 2.3 订单与产品要分清对象
- 产品页看的是产品模板
- 订单页看的是订单实例
- 订单里点商品名称，打开的是来源产品详情

### 2.4 规格与自动报价必须有独立区块
它不能埋在备注里，也不能散在多个字段里。

## 3.3 订单详情页

1. 顶部订单概览卡
2. 订单级信息卡组

- 客户姓名

- 剩余时间 / 超时

- 当前负责人

这是订单详情页第一屏最重要的内容。

---

### 3.3.2 订单级信息卡组
建议拆成 3 张信息卡：

#### 卡 1：客户与平台信息
- 客户姓名
- 联系方式

- 平台原始地址
- 客户备注

- 平台付款时间
- 客户期望时间

- 财务摘要（首轮可占位）
- 物流摘要（首轮可占位）
- 售后摘要（首轮可占位）
- 整单备注

---

### 3.3.3 商品协同区
这是订单详情页最核心的部分。

每个商品必须是一张独立商品卡。

#### 商品卡从上到下结构
1. 商品卡头部
2. 来源产品条
3. 规格与报价区
4. 客服需求区
5. 设计建模区
6. 跟单委外区
7. 工厂回传区

---

#### 商品卡头部
至少包含：

- 商品名称
- 数量
- 商品状态
- 优先级 / 加急（后补）
- 操作区（后补）

注意：  
**商品名称可点击，点击后打开来源产品详情抽屉。**

---

#### 来源产品条
必须包含：

- 来源产品名称
- 来源产品编号
- 来源版本信息
- 按钮：查看来源产品

用途：
- 说明当前商品引用自哪个产品模板
- 区分模板与实例

---

#### 规格与报价区
这是首轮必须重点实现的区块。

从上到下建议包含：

1. 规格选择器
2. 规格参数摘要
3. 基础价格卡
4. 附加价格列表
5. 系统参考报价卡
6. warning / conflict 提示区

- 标签或下拉
- 规格名称来自产品配置
- 如果产品规格必选，未选规格前必须提示

##### 规格参数摘要

###### 戒指
- 圈号
- 面宽
- 底宽
- 面厚
- 底厚
- 参考重量

###### 吊坠
- 档位
- 长
- 宽
- 厚
- 参考重量

##### 基础价格卡
显示：

- 当前规格基础价格

##### 附加价格列表
显示生效的：

- 材质加价
- 工艺加价
- 特殊需求加价

##### 系统参考报价卡
显示：

- 系统参考报价合计

##### 提示区
至少提示：

- 未选规格
- 当前规格没有基础价格
- 当前附加项没有对应价格规则

# 4. 来源产品详情抽屉结构

## 4.1 触发方式
在订单页点击商品名称，或点击“查看来源产品”。

## 4.2 从上到下结构
1. 标题区
2. 版本与来源提示卡
3. 标签页
4. 标签页内容
5. 底部操作区

---

## 4.3 标题区
包含：

- 标题：来源产品详情
- 产品名称
- 产品编号
- 关闭按钮

---

## 4.4 版本与来源提示卡
建议展示：

- 来源产品名称
- 引用时版本
- 当前产品版本（可后补）
- 引用时间（可后补）
- 引用人（可后补）

首轮至少要有“这是来源产品，不是订单商品”的明确提示。

---

## 4.5 标签页
首轮建议 3 个标签页：

1. 产品原始详情
2. 订单参数对比
3. 文件资料

---

## 4.6 标签页一：产品原始详情
建议结构：

1. 产品摘要卡
2. 基础信息
3. 参数配置
4. 定制规则
5. 生产参考

至少展示：

- 主图
- 名称 / 编号 / 品类 / 状态
- 支持材质 / 默认材质
- 支持工艺 / 默认工艺
- 特殊需求支持项
- 定制规则摘要
- 生产参考摘要

---

## 4.7 标签页二：订单参数对比
用于核对：

- 材质
- 工艺
- 规格值
- 是否需测量工具
- 是否需重新建模

建议采用对比表结构：

- 字段名
- 模板值
- 订单值
- 对比结果

结果可分为：

- 一致
- 已调整
- 冲突
- 待确认

首轮可先做轻量版。

---

## 4.8 标签页三：文件资料
建议展示：

- 来源产品图片摘要
- 建模文件摘要
- 其他文件摘要

首轮以“查看”结构为主，不做复杂文件管理。

---

## 4.9 底部操作区
建议至少保留：

- 关闭
- 在产品模块中查看

---

# 6. 首轮最重要的 UI 验收点

## 6.1 订单页不是大表单
订单详情页必须是“概览 + 商品卡 + 记录区”的结构。

## 6.2 商品卡必须突出“规格与报价区”
自动带价是首轮主链路，不能埋在别的区块里。

## 6.3 来源产品详情必须与订单商品详情分开
点击商品名称后，看到的是产品模板原始信息，不是订单商品实例信息。

## 6.4 产品编辑页必须有规格明细区
否则无法支撑订单里的自动带价。

## 6.5 戒指与吊坠两个样例必须都能走通
这不是可选项，是首轮验收样例。
