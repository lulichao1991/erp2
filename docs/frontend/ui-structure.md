# 页面结构总览（Purchase + OrderLine）

## 1. 文档目标
本文件用于统一当前前端页面结构，明确主页面从上到下有哪些区块、区块之间的层级关系，以及哪些能力只是兼容或占位。

当前主线覆盖：

- 工作台
- 销售中心
- 购买记录
- 款式管理
- 客户中心
- 任务中心
- 生产跟进
- 设计 / 建模工作台
- 工厂协同中心
- 财务中心
- 仓库商品管理
- 管理看板
- 生产计划

并服务于这条业务链：

**款式维护
-> 销售引用款式
-> 选择规格
-> 自动带出参数
-> 自动带出价格
-> 查看来源款式详情**

---

## 2. 总体 UI 原则

### 2.1 OrderLine 是主操作对象
- 销售中心的一行代表一件商品
- 购买记录详情页中的一行或一张卡代表一件商品
- 单件商品的状态、设计、生产、物流和售后都应优先挂在 `OrderLine` 上

### 2.2 Purchase 是归组对象
- 购买记录页看的是一次购买中的公共信息
- 购买记录用于归组同一次购买下的多个销售
- 不把单件商品执行字段塞进购买记录公共信息区

### 2.3 Product 是模板对象
- 款式页看的是款式模板
- 销售引用款式后，保留 `ProductSnapshot`
- 点击销售中的来源款式入口，打开来源款式详情

### 2.4 legacy `/orders` 已删除
旧 `/orders` 页面不再访问，不作为当前主 UI 入口，也不在侧边栏中展示。

---

## 3. 销售中心 `/order-lines`

### 页面目标
快速搜索、筛选、查看和推进单件销售。

### 从上到下结构
1. 页面标题区
2. 快捷统计卡
3. 搜索与筛选区
4. 销售表格
5. 销售详情抽屉

### 页面标题区
包含：
- 页面标题：销售中心
- 右侧按钮：新建购买记录

### 快捷统计卡
建议包含：
- 全部销售
- 生产中
- 待发货
- 售后中
- 异常 / 风险

### 搜索与筛选区
建议支持：
- 搜索货号 / 款式名称 / 客户 / 购买记录 / 平台订单号
- 销售状态
- 当前负责人

### 销售表格
建议字段：
- 货号、款式名称、版本号
- 商品缩略图
- 客户姓名、客户 ID
- 当前不展示或维护额外销售编号；所属购买记录通过购买记录编号和平台订单号追溯
- 当前状态
- 当前负责人
- 承诺交期
- 物流摘要
- 售后摘要
- 操作列

说明：
- 一行就是一件商品
- 即使同一个客户同一次购买有多件商品，也显示为多行
- 购买记录编号用于归组，不作为主操作对象

### 销售详情抽屉
建议区块：
1. 顶部摘要
2. 价格与财务
3. 状态更新
4. 操作日志
5. 定制参数
6. 来源款式详情
7. 规格参数
8. 设计 / 委外 / 生产信息
9. 物流记录
10. 售后记录
11. 所属购买记录入口

说明：
- 物流和售后默认关联 `orderLineId`
- 销售详情抽屉是单件商品执行信息的主要承载处
- 顶部摘要只展示识别、客户、负责人和关键时间信息；价格、定金、尾款和财务核算状态独立放在“价格与财务”区块。
- 跟单备注在“跟单 / 下厂”区块维护，工厂备注在“工厂回传”区块维护；“定制参数”不再承载生产备注。

## 4. 新建购买记录 `/purchases/new`

### 页面目标
创建一笔购买记录草稿，并逐件添加销售。

### 从上到下结构
1. 页面标题区
2. 购买公共信息区
3. 客户与收件信息区
4. 付款汇总区
5. 销售区域

### 页面标题区
包含：
- 页面标题：新建购买记录
- 返回销售中心
- 保存草稿
- 保存草稿允许资料暂缺，资料完整度只用于客服确认完成动作

### 购买公共信息区
首轮至少包括：
- 来源渠道
- 平台订单编号
- 付款时间
- 负责人
- 备注

### 客户与收件信息区
首轮至少包括：
- 客户姓名
- 联系方式
- 收件人
- 收件电话
- 收件地址

### 付款汇总区
当前只做购买记录层面的付款汇总。财务中心已经作为独立角色入口存在，但新建购买记录页不承接复杂对账、发票或收退款流水。

### 销售区域
这是新建购买记录页的核心区域。

每张销售草稿卡代表一件商品，至少支持：
- 引用款式模板
- 自动带出款式名称、品类、默认材质、默认工艺
- 选择规格
- 自动带出规格参数
- 自动带出基础价格
- 叠加固定加价规则
- 生成系统参考报价

说明：
- 页面创建的是 `Purchase`
- 销售草稿后续应落为 `OrderLine`
- 不回退成旧 `order.items` 主线

## 5. 购买记录详情 `/purchases/:purchaseId`

### 页面目标
作为购买记录查询页与销售归组页。

### 从上到下结构
1. 页面标题区
2. 购买记录摘要
3. 客户信息区
4. 付款总览
5. 本次销售列表
6. 购买记录时间线
7. 销售详情抽屉

### 购买记录摘要
必须展示：
- 购买记录编号
- 平台订单编号
- 来源渠道
- 聚合状态
- 销售数量
- 当前负责人
- 风险标签

说明：
- 这里展示的是购买公共信息和聚合信息
- 只做汇总，不承担单件商品推进

### 客户信息区
展示本次购买相关客户和收件信息。

### 付款总览
展示购买记录层面的付款汇总。

说明：
- 购买记录详情页不承接复杂财务处理；财务确认、结算和异常标记进入 `/finance`
- 销售报价仍以 `OrderLine.quote` 为准

### 本次销售列表
必须直接展示本次购买下的所有销售，避免把多件商品折叠成一条摘要。

建议字段：
- 货号、款式名称、版本号
- 商品缩略图
- 规格摘要
- 销售状态
- 当前负责人
- 物流摘要
- 售后摘要
- 操作列

### 购买记录时间线
展示购买记录层面的归组事件。

说明：
- 销售操作日志应优先在销售详情抽屉中查看
- 购买记录时间线只承载公共事件和聚合事件

## 6. 款式管理 `/products`

### 页面目标
维护款式模板，让销售可以引用标准资料、规格和价格规则。

### 款式列表页
建议包含：
- 搜索与筛选
- 产品表格
- 可引用状态
- 价格摘要
- 新建款式入口

### 款式详情页
展示：
- 产品基础信息
- 参数配置
- 规格明细
- 固定加价规则
- 定制规则
- 生产参考
- 文件资料
- 引用记录入口（可占位）

说明：
- 款式详情展示的是模板，不是销售实例

### 产品编辑页
用于编辑款式模板资料、参数配置、规格明细、价格规则、定制规则、生产参考、图片与文件。

说明：
- 当前版本号只做只读展示；只有款式外观结构、设计稿或设计方案变化才应升级版本，补齐参数、价格、生产参考或文件资料不触发升版。
- 编辑已有款式时提供“设计版本”区块；点击“创建设计新版本”会在同一个 `Product` 下新增版本记录，不需要新建款式。

## 7. 来源款式详情抽屉

### 触发方式
在销售详情或销售草稿中点击来源款式相关入口。

### 从上到下结构
1. 标题区
2. 来源款式基础信息
3. 模板原始详情
4. 销售参数对比
5. 文件资料

### 销售参数对比
首轮至少对比：
- 材质
- 工艺
- 规格值
- 是否需测量工具
- 是否需重新建模

说明：
- 左侧是款式模板允许的范围
- 右侧是当前销售或草稿的实际选择

## 8. 角色工作台与支撑页

### `/`
工作台。

结构：
- 顶部快速入口
- 当前交付重点
- 最近推进事项
- 购买记录概览

边界：
- 只做概览和跳转，不承接销售、生产、财务或库存录入动作
- 具体执行回到对应主页面

### `/customers` 与 `/customers/:customerId`
客户中心与客户详情。

结构：
- 客户列表 / 客户基础信息
- 购买记录、销售和售后聚合
- 跳转到购买记录详情或销售中心

边界：
- 第一版为只读聚合和追溯入口
- 不扩展复杂 CRM、营销自动化或客户画像

### `/tasks` 与 `/tasks/:taskId`
任务中心与任务详情。

结构：
- 任务筛选和任务列表
- 任务详情基础信息
- 关联购买记录、关联销售和任务相关时间线

边界：
- 任务只作为协作提醒和跟进入口
- 任务状态更新不等于推进 `Purchase` 或 `OrderLine`
- 销售阶段推进仍回到 `OrderLine.lineStatus`

### `/production-follow-up`
生产跟进 / 跟单视图。

结构：
- 状态快捷统计
- 待审核、待下发、生产中、待回传、异常 / 逾期等视图
- 工厂计划、下发、阻塞退回和退回设计 / 建模操作

边界：
- 主语是 `OrderLine`
- 只基于 `lineStatus / productionStatus / factoryStatus / factoryPlannedDueDate` 推进
- 不写回旧订单模型

### `/design-modeling`
设计 / 建模工作台。

结构：
- 设计 / 建模任务统计
- 待设计、设计中、待建模、建模中、待修改和已完成视图
- 设计文件、建模文件和出蜡文件记录

边界：
- 只展示商品执行需求、文件记录和修改任务
- 不展示客户联系方式、收货地址或财务金额

### `/factory`
工厂协同中心。

结构：
- 工厂任务统计
- 待接收、生产中、待回传、已回传和异常视图
- 工厂接收、开工、完成、异常和生产回传表单

边界：
- 只展示分配给当前 mock 工厂的 `OrderLine` 生产资料
- 不展示客户隐私、销售价格、定金、尾款、利润或财务备注

### `/finance`
财务中心。

结构：
- 购买记录收款摘要
- 财务任务统计
- 尾款、工厂结算、成本确认、异常标记和财务锁定

边界：
- 可以查看金额、成本和工厂回传数据
- 不推进设计、建模或生产执行字段

### `/inventory`
仓库商品管理。

结构：
- 库存总览统计
- 库管快捷视图
- 库位汇总
- 库存筛选、库存台账和库存详情
- 入库、占用、释放、出库、报废、质检处置和盘点

边界：
- 主语是 `InventoryItem`
- 库存流转只更新库存资产台账和库存流转记录
- 不推进 `OrderLine.lineStatus`、生产、财务或售后状态

### `/management`
管理看板。

结构：
- 业务总览
- 销售状态分布
- 生产风险
- 财务概览
- 角色负载
- 工厂表现

边界：
- 只做经营和风险观察
- 不承接客服、跟单、工厂、财务或库存录入动作

### `/production-plan` 与 `/production-plan/:taskId`
生产计划与生产计划详情。

结构：
- 工厂生产计划列表
- 阶段统计
- 生产资料摘要
- 货号详情和来源款式 / 关联任务入口

边界：
- 视图数据只来自当前 `tasks + purchases + orderLines + products`
- 不读取旧订单模型

## 10. 验收标准

- 销售中心清楚表达“一行 = 一件商品”
- 新建购买记录页清楚表达“1 笔购买记录 + 多条销售”
- 购买记录详情页清楚表达“购买记录归组，销售执行”
- 款式详情页清楚表达“产品是模板”
- 角色工作台清楚表达“协作入口 / 角色视图”，不替代 `OrderLine` 主流程
- 物流、售后默认关联 `orderLineId`
- 旧 `/orders` 不再被写成当前主模块或兼容入口
