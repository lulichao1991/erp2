# 页面结构总览（Purchase + OrderLine）

## 1. 文档目标
本文件用于统一当前前端页面结构，明确主页面从上到下有哪些区块、区块之间的层级关系，以及哪些能力只是兼容或占位。

当前主线覆盖：

- 销售中心
- 购买记录
- 产品管理

并服务于这条业务链：

**产品维护
-> 销售引用产品
-> 选择规格
-> 自动带出参数
-> 自动带出价格
-> 查看来源产品详情**

---

## 2. 总体 UI 原则

### 2.1 OrderLine 是主操作对象
- 销售中心的一行代表一件商品
- 购买记录详情页中的一行或一张卡代表一件商品
- 单件商品的状态、设计、生产、物流和售后都应优先挂在 `OrderLine` 上

### 2.2 Purchase 是归组对象
- 购买记录页看的是一次购买中的公共信息
- 购买记录用于归组同一次购买下的多个销售
- 不把单件商品执行字段塞进购买记录公共信息区

### 2.3 Product 是模板对象
- 产品页看的是产品模板
- 销售引用产品后，保留 `ProductSnapshot`
- 点击销售中的来源产品入口，打开来源产品详情

### 2.4 legacy `/orders` 已删除
旧 `/orders` 页面不再访问，不作为当前主 UI 入口，也不在侧边栏中展示。

---

## 3. 销售中心 `/order-lines`

### 页面目标
快速搜索、筛选、查看和推进单件销售。

### 从上到下结构
1. 页面标题区
2. 快捷统计卡
3. 搜索与筛选区
4. 销售表格
5. 销售详情抽屉

### 页面标题区
包含：
- 页面标题：销售中心
- 右侧按钮：新建购买记录

### 快捷统计卡
建议包含：
- 全部销售
- 生产中
- 待发货
- 售后中
- 异常 / 风险

### 搜索与筛选区
建议支持：
- 搜索货号 / 商品名称 / 客户 / 内部销售编号 / 购买记录
- 销售状态
- 当前负责人

### 销售表格
建议字段：
- 货号
- 商品缩略图、商品名称、版本号
- 客户姓名、客户 ID
- 内部销售编号 / 所属购买记录仅在详情或搜索上下文中使用，预览表格不展示
- 当前状态
- 当前负责人
- 承诺交期
- 物流摘要
- 售后摘要
- 操作列

说明：
- 一行就是一件商品
- 即使同一个客户同一次购买有多件商品，也显示为多行
- 购买记录编号用于归组，不作为主操作对象

### 销售详情抽屉
建议区块：
1. 顶部摘要
2. 状态更新
3. 操作日志
4. 销售基础信息 / 实际需求
5. 来源产品详情
6. 设计 / 委外 / 生产信息
7. 物流记录
8. 售后记录
9. 所属购买记录入口

说明：
- 物流和售后默认关联 `orderLineId`
- 销售详情抽屉是单件商品执行信息的主要承载处

## 4. 新建购买记录 `/purchases/new`

### 页面目标
创建一笔购买记录草稿，并逐件添加销售。

### 从上到下结构
1. 页面标题区
2. 购买公共信息区
3. 客户与收件信息区
4. 付款汇总区
5. 销售区域

### 页面标题区
包含：
- 页面标题：新建购买记录
- 返回销售中心
- 保存草稿

### 购买公共信息区
首轮至少包括：
- 来源渠道
- 平台订单编号
- 付款时间
- 负责人
- 备注

### 客户与收件信息区
首轮至少包括：
- 客户姓名
- 联系方式
- 收件人
- 收件电话
- 收件地址

### 付款汇总区
当前只做购买记录层面的付款汇总，不扩展复杂财务中心。

### 销售区域
这是新建购买记录页的核心区域。

每张销售草稿卡代表一件商品，至少支持：
- 引用产品模板
- 自动带出商品名称、品类、默认材质、默认工艺
- 选择规格
- 自动带出规格参数
- 自动带出基础价格
- 叠加固定加价规则
- 生成系统参考报价

说明：
- 页面创建的是 `Purchase`
- 销售草稿后续应落为 `OrderLine`
- 不回退成旧 `order.items` 主线

## 5. 购买记录详情 `/purchases/:purchaseId`

### 页面目标
作为购买记录查询页与销售归组页。

### 从上到下结构
1. 页面标题区
2. 购买记录摘要
3. 客户信息区
4. 付款总览
5. 本次销售列表
6. 购买记录时间线
7. 销售详情抽屉

### 购买记录摘要
必须展示：
- 购买记录编号
- 平台订单编号
- 来源渠道
- 聚合状态
- 销售数量
- 当前负责人
- 风险标签

说明：
- 这里展示的是购买公共信息和聚合信息
- 只做汇总，不承担单件商品推进

### 客户信息区
展示本次购买相关客户和收件信息。

### 付款总览
展示购买记录层面的付款汇总。

说明：
- 当前阶段不扩展复杂财务中心
- 销售报价仍以 `OrderLine.quote` 为准

### 本次销售列表
必须直接展示本次购买下的所有销售，避免把多件商品折叠成一条摘要。

建议字段：
- 销售编号
- 商品名称
- 规格摘要
- 销售状态
- 当前负责人
- 物流摘要
- 售后摘要
- 操作列

### 购买记录时间线
展示购买记录层面的归组事件。

说明：
- 销售操作日志应优先在销售详情抽屉中查看
- 购买记录时间线只承载公共事件和聚合事件

## 6. 产品管理 `/products`

### 页面目标
维护产品模板，让销售可以引用标准资料、规格和价格规则。

### 产品列表页
建议包含：
- 搜索与筛选
- 产品表格
- 可引用状态
- 价格摘要
- 新建产品入口

### 产品详情页
展示：
- 产品基础信息
- 参数配置
- 规格明细
- 固定加价规则
- 定制规则
- 生产参考
- 文件资料
- 引用记录入口（可占位）

说明：
- 产品详情展示的是模板，不是销售实例

### 产品编辑页
用于编辑产品模板和价格规则。

---

## 7. 来源产品详情抽屉

### 触发方式
在销售详情或销售草稿中点击来源产品相关入口。

### 从上到下结构
1. 标题区
2. 来源产品基础信息
3. 模板原始详情
4. 销售参数对比
5. 文件资料

### 销售参数对比
首轮至少对比：
- 材质
- 工艺
- 规格值
- 是否需测量工具
- 是否需重新建模

说明：
- 左侧是产品模板允许的范围
- 右侧是当前销售或草稿的实际选择

---

## 8. legacy `/orders` 删除状态

旧 `/orders` 相关页面、组件、service、mock 和类型已经删除。

说明：
- 当前主流程入口是 `/order-lines`、`/purchases/*`、`/customers/*`、`/tasks` 和 `/production-plan`
- 旧实现如需回看，使用 git 历史 / 删除前 PR

---

## 9. 验收标准

- 销售中心清楚表达“一行 = 一件商品”
- 新建购买记录页清楚表达“1 笔购买记录 + 多条销售”
- 购买记录详情页清楚表达“购买记录归组，销售执行”
- 产品详情页清楚表达“产品是模板”
- 物流、售后默认关联 `orderLineId`
- 旧 `/orders` 不再被写成当前主模块或兼容入口
