~~~md
# 前端布局框架与导航结构草案

## 1. 文档目标
本文件用于统一首轮前端后台壳结构，明确：

- 后台整体布局长什么样
- 左侧导航怎么组织
- 顶部栏承担什么功能
- 页面头部怎么统一
- 主内容区怎么排
- 哪些能力用页面，哪些能力用弹窗 / 抽屉

目标是让订单中心和产品管理在同一套后台框架下运行，不出现“每个页面像不同系统”的情况。

---

## 2. 总体框架

首轮建议采用标准中后台三段式结构：

1. 左侧侧边栏
2. 顶部栏
3. 主内容区

页面关系如下：

```text
AppLayout
├─ AppSidebar
├─ MainArea
│  ├─ AppTopbar
│  └─ PageContainer
│     ├─ AppBreadcrumb
│     ├─ PageHeader
│     └─ PageContent
~~~

------

## 3. 布局原则

### 3.1 以桌面端优先

首轮先按桌面端后台设计，不优先考虑复杂响应式折叠策略。

### 3.2 框架要稳，不追求花哨

当前重点是：

- 清晰
- 稳定
- 可扩展
- 适合信息密集页面

不是做营销官网。

### 3.3 页面头部统一

订单页、产品页都必须使用同一套：

- 面包屑
- 页面标题
- 页面操作区

### 3.4 弹窗和抽屉依附页面上下文

很多操作不适合做独立页面，首轮统一走页面上下文弹窗 / 抽屉。

------

## 4. 左侧侧边栏（AppSidebar）

## 4.1 首轮一级菜单

首轮只建议保留这些一级菜单：

- 工作台
- 订单中心
- 产品管理

不要一开始就把任务中心、文件中心、设置页全部铺开，否则前端壳子会显得很重。

------

## 4.2 菜单对应页面

### 工作台

首轮可先做占位入口，不作为重点开发页面。

### 订单中心

对应：

- `/orders`
- `/orders/new`
- `/orders/:orderId`

### 产品管理

对应：

- `/products`
- `/products/new`
- `/products/:productId`
- `/products/:productId/edit`

------

## 4.3 侧边栏视觉规则

- 宽度固定
- 激活菜单明确高亮
- 不做复杂多层级展开
- 图标 + 文字即可
- 当前模块状态明显

------

## 5. 顶部栏（AppTopbar）

## 5.1 当前阶段职责

顶部栏首轮主要承担：

- 页面级辅助信息
- 预留全局操作入口
- 保持后台整体统一性

------

## 5.2 首轮建议内容

左侧：

- 当前一级模块名称（可弱化）

右侧：

- 预留用户区（可先占位）
- 预留通知入口（可先占位）

首轮不要在顶部栏塞太多功能。

------

## 5.3 不建议首轮做的内容

- 全局复杂搜索
- 消息中心
- 多工作区切换
- 复杂筛选在 topbar 层统一承载

这些都容易把壳子做重。

------

## 6. 主内容区（MainArea）

## 6.1 结构

主内容区建议统一为：

1. 面包屑
2. 页面标题区
3. 页面正文区

------

## 6.2 页面宽度策略

建议采用：

- 整体页面铺满
- 内容容器留合理左右边距
- 表格页宽一些
- 表单页 / 抽屉页不必过窄

订单详情页和产品详情页都属于信息密度较高页面，要避免容器过窄导致内容挤压。

------

## 7. 面包屑（AppBreadcrumb）

## 7.1 作用

用于明确当前页面处于哪个模块和层级。

------

## 7.2 建议规则

### 订单模块

#### 订单列表页

- 订单中心

#### 新建订单页

- 订单中心 / 新建订单

#### 订单详情页

- 订单中心 / 订单详情 / {订单编号}

------

### 产品模块

#### 产品列表页

- 产品管理

#### 新建产品页

- 产品管理 / 新建产品

#### 产品详情页

- 产品管理 / 产品详情 / {产品名称}

#### 产品编辑页

- 产品管理 / 产品详情 / {产品名称} / 编辑产品

------

## 7.3 首轮要求

- 面包屑必须轻量
- 不做复杂下拉
- 不承担筛选功能

------

## 8. 页面头部（PageHeader）

## 8.1 统一结构

每个主页面都必须使用统一的头部结构：

### 左侧

- 页面标题
- 页面副标题（可选）

### 右侧

- 页面主按钮区

------

## 8.2 页面头部示例

### 订单列表页

- 标题：订单中心
- 右侧按钮：新建订单

### 产品列表页

- 标题：产品管理
- 右侧按钮：新建产品

### 订单详情页

- 标题：订单详情
- 右侧按钮：编辑订单 / 上传附件（后补）

### 产品详情页

- 标题：产品详情
- 右侧按钮：编辑产品

------

## 8.3 页面头部不要承载的内容

首轮不要在 `PageHeader` 里塞：

- 大量筛选器
- 二级 tab
- 大量批量操作

这些应放到页面正文区的专属区域里。

------

## 9. 页面正文区组织规则

## 9.1 订单模块

### 订单列表页

建议结构：

1. 快捷统计卡
2. 筛选区
3. 列表区

### 新建订单页

建议结构：

1. 基础信息
2. 客户信息
3. 时间信息
4. 商品明细

### 订单详情页

建议结构：

1. 顶部概览卡
2. 订单级信息卡组
3. 商品协同区
4. 记录区（物流 / 售后 / 附件 / 日志）

------

## 9.2 产品模块

### 产品列表页

建议结构：

1. 快捷统计卡
2. 筛选区
3. 列表区

### 产品详情页

建议结构：

1. 顶部摘要区
2. 分区导航
3. 分区内容区

### 产品编辑页

建议结构：

1. 顶部保存区
2. 左侧分区导航
3. 右侧表单区

------

## 10. 弹窗与抽屉规则

当前项目里，哪些能力应该用弹窗、哪些应该用抽屉，必须统一，不要随页面随意变化。

------

## 10.1 用大弹窗（LargeModal）的场景

### 产品引用选择器

必须使用大号弹窗。

原因：

- 需要左侧筛选
- 需要中间卡片网格
- 需要右侧预览
- 需要在订单上下文中完成，不适合独立页面跳转

建议宽度：

- `1280 ~ 1440px`

------

## 10.2 用右侧抽屉（SideDrawer）的场景

### 来源产品详情

必须使用右侧抽屉。

原因：

- 它属于订单页的上下文核对工具
- 看完需要快速回到订单
- 不适合跳到独立页面作为主流程

建议宽度：

- `640 ~ 760px`

------

## 10.3 用小确认框（ConfirmDialog）的场景

适用于：

- 删除
- 停用
- 放弃修改
- 替换模板确认（后补）
- 取消引用确认（后补）

------

## 10.4 首轮暂不建议单独做页面的能力

- 产品引用选择器
- 来源产品详情
- 物流编辑
- 售后编辑
- 新增价格规则

首轮优先使用弹窗 / 抽屉承载。

------

## 11. 页面与上下文弹层的关系

## 11.1 订单页面上下文能力

订单页是当前最复杂的工作台，因此允许挂载多个上下文弹层：

### 订单详情页可打开

- 产品引用选择器
- 来源产品详情抽屉
- 物流记录弹窗（后补）
- 售后记录弹窗（后补）
- 商品信息编辑抽屉（后补）

------

## 11.2 产品页面上下文能力

产品页当前首轮较简单：

### 产品详情页可打开

- 上传文件抽屉（后补）
- 引用记录抽屉（后补）
- 版本记录抽屉（后补）

### 产品编辑页可打开

- 新增价格规则弹窗（后补）

------

## 12. 当前首轮推荐页面密度

### 订单详情页

信息密度高，但必须分区明确。

重点是：

- 商品卡
- 规格与报价区
- 来源产品核对能力

### 产品详情页

信息密度中等，重点是：

- 顶部摘要
- 参数配置
- 价格规则
- 规格明细
- 文件摘要

### 产品编辑页

操作密度高，重点是：

- 左右分区
- 表单分组
- 规格表和规则表

------

## 13. 首轮最小导航闭环

首轮必须至少保证下面这条导航闭环能跑通：

### 订单侧

订单列表
→ 订单详情
→ 商品卡中引用产品
→ 打开产品引用选择器
→ 完成引用
→ 回到订单详情
→ 点击商品名称
→ 打开来源产品详情抽屉
→ 从抽屉跳转到产品详情

### 产品侧

产品列表
→ 产品详情
→ 产品编辑
→ 返回产品详情
→ 返回产品列表

------

## 14. 推荐实现顺序（布局层面）

### 第一步

先做：

- AppLayout
- Sidebar
- Topbar
- Breadcrumb
- PageHeader

### 第二步

再做：

- 订单和产品的列表页壳
- 详情页壳
- 编辑页壳

### 第三步

再接：

- 弹窗
- 抽屉
- 页面上下文动作

------

## 15. 首轮验收标准

### 15.1 结构统一

订单中心和产品管理必须看起来属于同一个系统。

### 15.2 页面关系清晰

用户能通过侧边栏、列表、详情、抽屉完成清晰跳转。

### 15.3 主页面少但够用

不依赖大量独立页面，也能完成核心业务链路。

### 15.4 上下文体验顺畅

订单页里的产品引用与来源产品核对，不应打断用户主流程。
