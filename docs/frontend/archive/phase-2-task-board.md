# Phase 2 Task Board（第二轮开发任务拆解清单）

## 1. 文档定位

本文档用于把 Phase 2 的产品规划拆解成可执行的前端开发任务清单。

本任务板只服务当前仓库这一轮目标，不扩散到完整 ERP 大版本。  
Phase 2 的核心范围只有三块：

- 订单中心升级为业务流转中心
- 新增任务中心
- 产品管理升级为产品标准档案

本文档适合用于：

- 给 AI coding 工具执行
- 给开发者排优先级
- 给自己做阶段检查
- 与 `phase-2-prd.md`、`order-lifecycle-spec.md`、`task-center-prd.md`、`role-draft.md` 联动使用

---

## 2. Phase 2 总目标

本轮的交付目标不是“多做几个页面”，而是让当前系统从：

**前端展示版**

升级为：

**业务流转版**

Phase 2 结束后，系统需要具备以下能力：

1. 订单具备明确状态体系
2. 一个订单可挂多个订单商品
3. 订单详情能看到报价调整与时间线
4. 能从订单发起任务
5. 有任务列表页和任务详情页
6. 产品管理能承载更完整标准资料
7. 首页能展示轻量业务摘要

---

## 3. 执行原则

### 原则 1：先补数据结构，再改页面
不要先写页面硬编码，再回头补类型和 mock。

### 原则 2：先升级订单详情页，再加任务中心
任务中心依赖订单来源，所以先把订单对象和订单详情页结构立住。

### 原则 3：先保证主链路可用，再补角色差异
先让“产品 → 订单 → 任务”跑通，再做角色视图差异。

### 原则 4：先做可展示和可联动，再做精细化交互
例如：
- 先有任务列表
- 再做复杂筛选
- 再做细节优化

### 原则 5：不提前做后端化复杂设计
本轮仍以 mock、前端状态管理、页面联动为主，不提前引入完整接口层复杂度。

---

## 4. 任务拆解总览

Phase 2 建议拆成 8 个任务包：

- A. 文档与类型对齐
- B. Mock 数据升级
- C. 订单中心升级
- D. 任务中心新增
- E. 产品管理升级
- F. 首页增强
- G. 角色模式接入
- H. 收尾与联调

建议执行顺序：

A → B → C → D → E → F → G → H
